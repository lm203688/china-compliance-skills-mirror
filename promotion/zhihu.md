# 给AI Agent装上"中国合规大脑"：4个开源Skill解决内容合规痛点

## 背景

做中文内容运营的都知道，合规是最大的坑：

- "最好"两个字，广告法罚款20-100万
- 用户数据传到海外服务器，没做安全评估？违法
- AI生成的内容不标注？2025新规已生效

大多数AI Agent（Claude Code、Cursor、OpenClaw等）不知道这些规则。它们会帮你写"全网最好"的文案，但不会告诉你这是违法的。

## 解决方案

我做了4个开源AI Agent Skill，让任何AI Agent都能具备中国合规专业知识：

### 1. 🛡️ cn-compliance-guard — 内容合规守卫
- 200+违禁词数据库，6个法律类别
- 5个平台SEO规则（百度/小红书/抖音/淘宝/京东）
- 自动提供安全替换建议

**实测**：一条抖音脚本，3个人工审核没发现问题，Skill扫出23个违禁词。

### 2. 🔍 cn-ai-visibility — AI搜索可见度检测
- 覆盖5个中国AI搜索引擎（DeepSeek/Kimi/豆包/通义千问/文心一言）
- 每个引擎的引用逻辑分析
- 针对性优化策略

**实测**：帮助一个SaaS品牌30天内AI引用率从0%提升到47%。

### 3. 🛡️ cn-data-export — 数据出境合规评估
- 7问风险评估
- 覆盖PIPL/网络安全法/数据安全法
- 自动推荐合规路径（安全评估/标准合同/认证）

### 4. 🤖 cn-aigc-detector — AI生成内容检测
- 5项统计信号分析
- 段落级可疑标注
- 适合审核UGC内容

## 为什么不是"又一个prompt"？

参考ConardLi/garden-skills（6.4K⭐）的设计哲学，每个Skill包含：
- **方法论**：为什么这样检测，法律依据是什么
- **检测流程**：结构化的步骤，不是模糊指令
- **结果解读**：输出格式标准化，可被程序解析
